1. What this app does

Scriptilyx SE is a visual script builder for Space Engineers programmable block scripts. Instead of writing C# by hand, you place nodes on a graph, connect them in order, set their properties, and generate a Space Engineers script.

For beginners

You can build simple scripts like “press a button, turn on a light, wait, turn it off” without knowing C#.

For advanced users

You can use command routing, variables, sections, custom code, generic terminal actions, and imported API dump nodes.

Important: Scriptilyx SE creates scripts for the Space Engineers programmable block. You still paste the generated script into Space Engineers and run it from a programmable block, button panel, timer block, or cockpit toolbar.

4. Node basics

A node is one step in your script. Examples:

  • Start begins the script.
  • Light On turns a light on.
  • Wait Seconds pauses before moving to the next node.
  • Set LCD Text writes text to a screen.
  • Stop Script ends the script.

Ports

Ports are the connection points on a node.

Port typeMeaningExample
Input portWhere the script enters the node.In
Output portWhere the script leaves the node.Next, True, False, Loop, Done
Connection rule: One output should only go to one place. One input can accept multiple incoming connections, so several branches can merge into one Wait node or Stop Script node.

7. Generate and use the script

  1. Build your graph.
  2. Click Check or Advanced Validate.
  3. Click the top ▶ Generate Serious Script button.
  4. Review the generated script in the output panel.
  5. Click Copy or Export Generated Script.
  6. Open Space Engineers.
  7. Open your programmable block.
  8. Paste the generated script.
  9. Click Check Code.
  10. Click Remember & Exit.
  11. Run the programmable block.
Important: The block names in the node properties must match the real Space Engineers block names exactly. If the node says Test Light, the block in Space Engineers must also be named Test Light.

9. Node Database Manager

The Node Database Manager edits the app’s node library. This is where nodes are stored before you add them to the graph.

What the Node Database Manager can do

  • Add new node definitions.
  • Edit node titles, descriptions, categories, ports, properties, previews, and ActionTypes.
  • Delete nodes.
  • Export selected nodes to JSON as backups.
  • Import node JSON files back into the database.
  • Duplicate selected nodes.

ActionType vs ActionId vs PropertyId

11. Space Engineers API dump import

The Import Space Engineers API Dump button imports a JSON file made from Space Engineers. It can create imported nodes from block actions and properties.

What the API dump contains

{
  "Blocks": [
    {
      "TypeName": "MyBatteryBlock",
      "DisplayName": "Battery",
      "CustomName": "Battery",
      "Actions": [
        { "Id": "OnOff_On", "Name": "Toggle block On" }
      ],
      "Properties": [
        { "Id": "OnOff", "TypeName": "Boolean" }
      ]
    }
  ]
}

What happens when you import it

Dump itemImported node usesGenerator support
ActionsApplyTerminalActionSupported
Boolean propertiesSetTerminalBoolSupported
Single / float propertiesSetTerminalFloatSupported
String / StringBuilder propertiesSetTerminalStringSupported
Int / Int64 propertiesSetTerminalIntSupported when the property accepts it
Color propertiesMay be skipped or need custom supportNot generally supported by the current generic nodes
Does the generator need an update? Usually no, as long as the imported nodes use the generic ActionTypes above. The generator only needs updates for brand-new special logic nodes that cannot be done with generic terminal actions or properties.

Beginner testing order

  1. Test Apply Any Terminal Action with a light and OnOff_On.
  2. Test Set Any Bool Property with a light and OnOff.
  3. Test Set Any Float Property with a light and Radius.
  4. Test Set Any Text Property on a block you do not mind renaming.
  5. Test Set Any Int Property only when you have a real int property from the dump.

13. Scanner instructions

The app can create a Space Engineers scanner script. The scanner is used to get exact ActionId and PropertyId values from your actual Space Engineers world.

This guide does not include the API dump script code. Use the app’s 📡 Create Space Engineers Scanner Script button to get the scanner script, or use the separate SpaceEngineersApiDumpScanner.cs file if you have it.

How to create an importable API dump JSON

  1. Click 📡 Create Space Engineers Scanner Script in Scriptilyx SE.
  2. Copy the scanner script.
  3. Open Space Engineers.
  4. Place or open a programmable block.
  5. Paste the scanner script into the programmable block.
  6. Click Check Code.
  7. Click Remember & Exit.
  8. Run the programmable block.
  9. Open the programmable block’s Custom Data.
  10. Press Ctrl + A, then Ctrl + C.
  11. Paste into Notepad.
  12. Save it as SpaceEngineers_API_Dump.json.
  13. Go back to Scriptilyx SE.
  14. Click ⬇ Import Space Engineers API Dump.
  15. Select SpaceEngineers_API_Dump.json.

Recommended scanner arguments

ArgumentUse it for
uniqueOne example per block API type. Best for a clean import.
filter LightOnly light-related blocks.
filter TimerOnly timer-related blocks.
filter RotorOnly rotor-related blocks.
allblocksEvery visible block. This can be very large.

15. Sections and subroutines

Sections let you create reusable chunks of graph logic.

NodePurpose
Start SectionMarks the beginning of a reusable section.
Call SectionRuns a named section and returns back when done.
ReturnEnds the section and returns to Call Section.Next.

Correct pattern

Main graph:
Start
→ Call Section
   SectionName = TestSection
→ Set LCD Text "RETURNED FROM SECTION"
→ Stop Script

Section graph:
Start Section
  SectionName = TestSection
→ Set LCD Text "SECTION RAN"
→ Wait Seconds
→ Return
Remember: The Return node goes back to whatever is connected to Call Section.Next.

Space Engineers setup

  2. Set a button action to Run Programmable Block.
  3. Set the argument to something like button_1.

Graph pattern

   Argument = button_1

17. Repeat and router nodes

Repeat Times

Repeat Times loops a path a fixed number of times.

Repeat Times.In starts the loop.
Repeat Times.Loop runs each loop.
Repeat Times.Repeat is where the loop path returns.
Repeat Times.Done runs after all loops are complete.

Number Greater Router

Number Greater Router checks a number variable and sends the script to one output based on how high the number is.

count > 6 → Greater6
count > 5 → Greater5
count > 4 → Greater4
count > 3 → Greater3
count > 2 → Greater2
otherwise → Else
For a light progress test that starts at count = 0 and adds 1 each loop, use Repeat Times = 7 if you want Greater6 to trigger.

21. Troubleshooting

ProblemLikely causeFix
Script says missing blockThe block name in the node does not match Space Engineers.Rename the block or update the node’s BlockName.
Unsupported ActionTypeThe generator does not know that ActionType.Use a supported generic ActionType or update the generator.
Button Command does nothingThe programmable block argument does not match.Check the button panel argument.
Section runs but does not returnCall Section.Next is not connected, or Return is missing.Connect Call Section.Next to the node that should run after the section.
Only some nodes runThe nodes may not be reachable from Start.Click Reachable Nodes.
Imported nodes do not workBlockName may be using a display name instead of the exact custom block name.Change BlockName to match the real block name in Space Engineers.
Merge block check is always falseThe merge block may not actually be connected, or the wrong block name is being used.Check the block name and make sure the merge block is physically connected to another merge block.
Sensor active check is always falseThe sensor may not be detecting the target type or the sensor range may be too small.Open the sensor settings in Space Engineers and check players, enemies, floating objects, range, and ownership settings.
Hinge moves the wrong directionThe RPM sign is reversed for the direction you expected.Use a negative VelocityRPM value, such as -5, instead of 5.
Hinge does not moveThe hinge may be locked, off, blocked by limits, or has too little torque.Try Hinge On, Hinge Rotor Lock Off, check limits, and test with a slow RPM.
Jump Drive charge check never becomes trueThe jump drive may not be charged enough, the wrong block name may be used, or recharge may be off.Check the block name, turn recharge on, and test with a lower percent such as 10%.
Jump Drive will not jump from scriptThe actual Jump action is not normally exposed to programmable block scripts.Use Scriptilyx SE to monitor/configure the jump drive, then trigger the actual jump from cockpit or toolbar controls.
Script did not save where expectedNo folder was selected in the Script Folders tab, or a different folder was selected.Select the folder in the tree first, then click Save beside Copy in the Script Preview area.
File name cannot be changedThis is intentional.The file name is locked to Script.cs so every saved Space Engineers script has the same expected file name.
Generic property does not workWrong PropertyId or wrong value type.Use the scanner/API dump to find the exact PropertyId and TypeName.

  4. Select the folder where you want the script saved.
  5. Click Save beside the Copy button in the Script Preview area.
  6. The app saves the Script Preview as Script.cs inside the selected folder.
Why the file is always Script.cs: The file name is locked so every saved programmable block script uses the same predictable name. This makes it easier to copy, replace, and organize scripts.
